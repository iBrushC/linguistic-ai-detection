# Linguistic AI Detection

This repo uses linguistic stylometry with human and AI authors to explore detecting AI by inversion: identifying an individual human author rather than trying to detect a general "AI" style.

## Why

It is widely known that "AI detectors" are extremely flawed, many of them being overly or underly sensitive. The problem is nearly impossible due to the sheer amount of different LLMs, how many personalize their conversation style to the user, and that a single essay (or worse paragraph) just cannot easily be attributed to anyone.

Despite that, there are many cases where it would be useful to detect AI-written works, primarily in education. Since detecting the ambiguous "AI" is a near-impossible challenge, what if we tried to detect individual authors and identify AI-generated writing by inversion?

## How

The basis is linguistic stylometry, something which has long been used for author attribution. Using various computed metrics, we compare an unknown piece of text to a known corpus. The current pipeline produces continuous similarity scores and author-similarity matrices; it does not yet apply a binary pass/fail threshold.

### Dataset

The ground-truth corpus is generated from the [Guardian Authorship dataset](https://huggingface.co/datasets/Efstathios/guardian_authorship), using the `cross_topic_1` configuration in `src/dataset.py`.

The dataset builder combines the `train`, `validation`, and `test` splits, uses seed `42`, and selects five articles per author for a total of 25 articles:

- Catherine Bennett
- George Monbiot
- Hugo Young
- Jonathan Freedland
- Martin Kettle

The selected articles cover the dataset's Politics, Society, UK, World, and Books topics. Running `python src/dataset.py` regenerates `essays.json`.

### Metrics

Each text is reduced to a set of per-sentence distributions across stylometric markers, then compared.

**Lexical**: sentence length, words per sentence, word length, type/token ratio (TTR), and corrected TTR (CTTR).

**Morphological**: POS-tag counts from NLTK, including adjective, adposition, adverb, auxiliary, subordinating conjunction, noun, particle, pronoun, and verb.

**Syntactical**: spaCy dependency counts for adverbial modifier, adjectival modifier, determiner, direct object, nominal subject, object of preposition, and root.

**Structural**: tricolon, cleft, normalization ("there is…"), conjunctions per coordinated series, and connective density split into additive, adversative, causal, and temporal.

The following metrics were identified as anti-signal and removed from the Brunner–Munzel aggregation: `dep_conjunct`, `existential_extraposition_counts`, `pos_CC`, `anadiplosis_counts`, `segments_per_sentence`, and `pos_JJS`. They remain available in the raw feature extractor, but are marked as red herrings and excluded when using the `brunnermunzel` method.

### Evaluation methods

The main evaluation is `author-similarity`, which calculates mean essay-pair similarities for each author pair. Diagonal cells represent within-author similarity, while off-diagonal cells represent cross-author similarity.

The available similarity methods are:

- `simple`: weighted aggregation of per-metric Kolmogorov–Smirnov similarities. Optional metric weights can be generated by `src/tune.py`.
- `manova`: sentence-level MANOVA using `1 - Pillai's trace` as the similarity score.
- `brunnermunzel`: Brunner–Munzel relative-effect similarities aggregated by metric family, with the red-herring metrics excluded.

The `single` command remains available as a two-author smoke test. It runs per-metric comparisons and produces diagnostic plots. `src/test_global_similarity.py` contains smoke tests for the similarity methods. AI-style essays can be generated by `src/generate.py`, but the current test harness does not score generated essays against the human corpus.

### Running the evaluations

```text
python src/dataset.py
python src/test.py author-similarity
python src/test.py author-similarity --method manova
python src/test.py author-similarity --method brunnermunzel
python src/test.py author-similarity --weights src/plots/metric_weights.json
python src/tune.py
python src/test_global_similarity.py
```

The author-similarity command writes plots and `author_similarity_matrix.json` to `src/plots/` by default.
